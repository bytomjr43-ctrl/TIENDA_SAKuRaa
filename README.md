Tienda_Sakura — Sistema de Gestión de Compras en Java

Tienda_Sakura es un proyecto desarrollado en Java bajo el perfil de estuddio de la materia Programación Orientada a Objetos. El objetivo es modelar una tienda virtual con un ecosistema de clases que representan clientes, productos, categorías, compras y administración del sistema.

Este proyecto forma parte de la materia Programación Orientada a Objetos, y está implementado completamente en Java aplicando conceptos de:

 -> Clases y objetos

 -> Encapsulamiento

 -> Herencia

 -> Clases abstractas e interfaces

 -> Agregación y composición

 -> Uso de colecciones genéricas (ArrayList)

 -> Manejo de relaciones entre objetos

 **Funcionalidades Principales**

 1. Gestión de clientes: creación, autenticación básica y métodos de pago.

 2. Carrito de compras: agregar, listar o eliminar productos antes de confirmar la compra.

 3. Simulación de compra: descuento automático de stock y registro en el historial del cliente.

 4. Gestión de fábricas y trabajadores: modela relaciones jerárquicas y registros internos.

 5.Jerarquía administrativa: permite interacción de roles como administradores y dueña (DuenaSakura).

 6.Persistencia en memoria: los datos se mantienen en ejecución usando listas (ArrayList).

Tiene una interacción en consola que muestra lo sieguiente:


  Ver catálogo de productos.
  Agregar producto al carrito.
  Ver carrito actual.
  Confirmar compra.
  Ver historial de compras.
  Salir.

  Este proyecto permitió aplicar de forma práctica los conceptos de **Programación Orientada a Objetos** junto con la **persistencia en bases de datos**.  
  Durante el desarrollo se implementaron clases, herencia, encapsulamiento y el patrón **DAO** para separar la lógica del acceso a datos.  
  Con esto se logró construir un sistema funcional capaz de registrar, listar y administrar la información de una tienda, mostrando cómo se puede conectar un programa Java con una base de datos real.
